🚀 PASOS PARA FUNCIONAR
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. CONFIGURAR FIREBASE
   - Ir a https://console.firebase.google.com
   - Crear proyecto "editorial-app"
   - Agregar Web App
   - Copiar config → .env.local
   - Habilitar Authentication (Email/Password + Google)
   - Crear Firestore Database (modo producción)
   - Aplicar firestore.rules

2. LLENAR .env.local
   - Pegar valores de Firebase
   - Cambiar VITE_ADMIN_EMAIL a TU_EMAIL

3. ACTUALIZAR firestore.rules
   - Cambiar 'TU_EMAIL_ADMIN' a TU_EMAIL (4 lugares)
   - Publicar reglas

4. INICIAR SERVIDOR
   npm run dev

5. TESTEAR
   - http://localhost:5173 → debe redirigir a /login
   - Registrar usuario → debe ir a /app
   - Editor debe funcionar igual
   - Login con admin email → acceso a /admin/*

⚠️ PROBLEMAS COMUNES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

"Firebase config not found"
  → .env.local debe estar en la RAÍZ de editorial-app/
  → Verificar que los valores no están vacíos
  → Reiniciar: npm run dev

"Cannot register user"
  → Verificar que Email/Password está habilitado en Firebase
  → Revisar DevTools → Console para más detalles

"Permission denied" en Firestore
  → firestore.rules debe tener email admin correcto
  → Publicar rules en Firebase Console
  → Esperar 1-2 minutos para que se apliquen

Admin no ve botón "Panel Admin"
  → Email en .env.local debe coincidir EXACTAMENTE
  → Logout y volver a login
  → Revisar mayúsculas/minúsculas
