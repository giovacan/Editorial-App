3 PROFESSIONAL TYPESETTING ALGORITHMS
═══════════════════════════════════════════════════════════════════════════════

ALGORITHM 1: PARAGRAPH BALANCING
Purpose: Avoid widow/orphan splits across pages

Function: balanceParagraphSplit()
Rules:
  - Minimum 2 lines per split side
  - Optimal ratio: 60% previous, 40% next
  - Detect violations and recommend actions

Example:
  Input: prevPage with 10 lines, nextPage with 1 line
  Output: { needsRebalance: true, action: 'move_line_backward' }
  Effect: Improves visual balance across page boundaries

Integration: applyFillPassInPlace() - 5 lines

---

ALGORITHM 2: PARAGRAPH COMPRESSION
Purpose: Remove small white gaps at page bottom invisibly

Function: tryParagraphCompression()
Rules:
  - Only compress paragraphs (not headers, lists, blockquotes)
  - Max line-height reduction: 4%
  - Max margin reduction: 20%
  - Invisible to reader

Example:
  Input: 12px gap at page bottom, 3 paragraphs
  Output: { canCompress: true, strategy: 'line_height_reduction', amount: '2%' }
  Effect: Fills gap, avoids unnecessary page breaks

Integration: processChapter() - 15 lines

---

ALGORITHM 3: GLOBAL LAYOUT OPTIMIZATION
Purpose: Choose optimal page breaks using quality scoring

Functions: evaluatePageQuality() + compareLayoutOptions()
Scoring:
  - White space penalty: remainingSpace * 0.5
  - Widow penalty: +50
  - Orphan penalty: +50
  - Heading at bottom penalty: +40
  - Lower score = better layout

Example:
  Input: Option A (break here), Option B (include element)
  Output: { recommended: 'B', reason: 'Option B better quality' }
  Effect: Chooses optimal break point, improves global layout

Integration: processChapter() - 20 lines
